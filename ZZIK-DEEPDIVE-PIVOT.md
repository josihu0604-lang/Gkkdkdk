## 🎯 핵심 질문 프레임워크

### 1. 사업 본질은 무엇인가?

#### Option A: **발견(Discovery) 플랫폼**
```
"외국인 관광객이 로컬 매장을 발견하게 돕는다"

예시:
├─ 小红书 (RED/Xiaohongshu) - 중국
├─ Tabelog - 일본
└─ Yelp - 미국

핵심: 콘텐츠 → 발견 → 방문
```

#### Option B: **전환(Conversion) 플랫폼**
```
"프랜차이즈가 외국인을 실제 매출로 전환하게 돕는다"

예시:
├─ 쿠팡이츠 (음식 전환)
├─ 야놀자 (숙박 전환)
└─ 카카오T (이동 전환)

핵심: 체험권 → 전환 → 매출
```

#### Option C: **하이브리드**
```
"콘텐츠로 발견 → GPS로 검증 → 체험권으로 전환"

예시:
├─ Foursquare + 쿠폰 (과거 모델)
├─ Swarm + 리워드
└─ 신규 카테고리 창출

핵심: 전체 퍼널 장악
```

## 📊 시장 벤치마킹 분석

### Case 1: 小红书 (RED/Xiaohongshu)

**모델:**
```
UGC 콘텐츠 플랫폼
├─ 이미지 + 텍스트 중심
├─ 위치 태그 (필수 아님)
├─ 큐레이션 알고리즘
└─ 커머스 연결 (구매 링크)

MAU: 3억+
수익: 광고 + 커머스 수수료
```

**ZZIK 적용 시:**
```
✅ 장점:
├─ 중국인 익숙한 UX
├─ 콘텐츠 재사용 쉬움
└─ 바이럴 효과 높음

❌ 단점:
├─ 콘텐츠 품질 통제 어려움
├─ GPS 검증 필요성 낮음
└─ 체험권 전환 약함
```

### Case 2: TikTok Local (실험 중)

**모델:**
```
위치 기반 숏폼 피드
├─ 세로 영상 (15-60초)
├─ "Near Me" 피드
├─ 비즈니스 프로필
└─ 광고 + 쿠폰 통합 테스트 중

현황: 미국 일부 도시 파일럿
```

**ZZIK 적용 시:**
```
✅ 장점:
├─ 영상 = 높은 전환율
├─ 실시간성 = 즉시 방문
└─ 젊은 층 인게이지먼트

❌ 단점:
├─ 영상 제작 진입장벽
├─ 인프라 비용 (CDN, 트랜스코딩)
└─ 콘텐츠 모더레이션 부담
```

### Case 3: Foursquare Swarm (쇠퇴)

**모델:**
```
체크인 게이미피케이션
├─ GPS 체크인
├─ 배지 + 메이요어
├─ 리워드 (초기)
└─ B2B 데이터 판매

현황: 사용자 급감 (게임만으론 부족)
```

**ZZIK 적용 시:**
```
✅ 장점:
├─ GPS 검증 = 핵심 기술
├─ 게이미피케이션 검증됨
└─ B2B 데이터 가치

❌ 단점:
├─ 체크인만으론 리텐션 약함
├─ 콘텐츠 없으면 바이럴 없음
└─ 수익화 어려움 (Foursquare 증명)

## 🔬 두 방향 상세 비교

### 방향 A: 실시간 릴스 지도

```
┌─────────────────────────────────────┐
│         [지도 뷰 - 메인]             │
│  📍 내 위치 중심                     │
│  ┌───┐ ┌───┐ ┌───┐                │
│  │🎥 │ │🎥 │ │🎥 │  <- 릴스 핀    │
│  └───┘ └───┘ └───┘                │
│                                     │
│  클릭 → 릴스 재생 → 스와이프         │
└─────────────────────────────────────┘

핵심 UX:
1. 앱 열면 지도 + 내 주변 릴스 핀
2. 핀 클릭 → 세로 풀스크린 릴스
3. 위/아래 스와이프 → 다른 매장 릴스
4. "지금 방문하기" → 체험권 활성화
5. 도착 → GPS 검증 → 체크인 → 사용
```

#### 사용자 여정

**관광객 (콘텐츠 소비자)**
```
09:00 홍대 도착
  ↓
앱 열기 → 지도에 주변 릴스 20개 표시
  ↓
"카페 릴스" 클릭 → 중국인이 찍은 영상
  ↓
"맛있어 보인다" → [지금 방문하기]
  ↓
도보 5분 → 도착 → GPS 자동 체크인
  ↓
체험권 사용 (아메리카노 무료)
  ↓
만족 → 본인도 릴스 촬영 → 업로드
```

**관광객 (콘텐츠 생성자)**
```
카페 방문 → 맛있음
  ↓
앱에서 "릴스 촬영" 버튼
  ↓
15초 영상 → GPS 위치 자동 태그
  ↓
업로드 → 무결성 검증
  ↓
지도에 실시간 표시 (30분간 강조)
  ↓
스탬프 적립 (10회 → 체험권 선택권)
```

**프랜차이즈 본사 시점**
```
월 50만원 결제
  ↓
ZZIK 대시보드 로그인
  ↓
실시간 메트릭:
├─ 우리 매장 릴스 업로드 수
├─ 릴스 뷰 수
├─ 릴스 → 방문 전환율
├─ 체험권 사용률
└─ 추가 구매율

인사이트:
"성수 A점 릴스 → 방문 85% (우수)"
"홍대 B점 릴스 → 방문 45% (개선 필요)"
```

#### 기술 스택

```
Frontend:
├─ React Native (iOS/Android)
├─ Mapbox GL (지도 + 커스텀 핀)
├─ React Native Video (릴스 재생)
└─ Gesture Handler (스와이프)

Backend:
├─ Node.js + Express
├─ PostgreSQL + PostGIS
├─ Redis (실시간 핀 캐싱)
└─ AWS MediaConvert (영상 트랜스코딩)

인프라:
├─ AWS S3 (영상 저장)
├─ CloudFront (CDN)
├─ Lambda@Edge (지역별 최적화)
└─ WebSocket (실시간 핀 업데이트)

비용 예상 (월):
├─ CDN: $500-2,000 (트래픽 기반)
├─ 트랜스코딩: $300-1,000
├─ 저장소: $100-500
└─ 서버: $200-800
총: $1,100-4,300/월
```

#### 장점 ✅

1. **차별화 명확**
   - 기존: 小红书 (피드) / TikTok (글로벌 피드)
   - ZZIK: 로컬 실시간 지도 + 릴스 (유일)

2. **즉시성**
   - "지금 이 순간" 경험 공유
   - 실시간 = 신뢰도 ↑
   - FOMO 효과 (빨리 가야 함)

3. **GPS 검증 필수성**
   - 지도 기반 = GPS 자연스러움
   - 가짜 콘텐츠 필터링
   - 실방문 증명 = B2B 가치

4. **발견 경험 최적화**
   - "주변에 뭐 있지?" → 지도 최적
   - 시각적 밀도 (핀 많음 = 인기 지역)
   - 탐색 재미 (숨은 맛집 발견)

5. **바이럴 구조**
   - 릴스 촬영 → 스탬프 → 체험권
   - 선순환 유도

#### 단점 ❌

1. **콜드 스타트 문제**
   - 초기: 지도에 핀 없음 = 가치 없음
   - 최소 임계: 지역당 릴스 50개 필요
   - 해결: 시드 콘텐츠 필요 (인플루언서 섭외)

2. **영상 제작 진입장벽**
   - 사진보다 어려움
   - 퀄리티 낮으면 전환 ↓
   - 해결: 템플릿 제공, 자동 편집

3. **인프라 비용**
   - CDN + 트랜스코딩 = 고정비 높음
   - 초기 사용자 적으면 비효율
   - 해결: 파일럿은 저해상도, 점진적 확장

4. **모더레이션 부담**
   - 부적절 영상 필터링
   - 가짜 리뷰
   - 해결: AI 필터 + 신고 시스템

5. **UX 복잡도**
   - 지도 + 릴스 + 체험권 = 학습 곡선
   - 해결: 온보딩 튜토리얼 강화

**관광객 (계획형)**
```
여행 전날 (중국에서)
  ↓
ZZIK 앱 다운 → 피드 탐색
  ↓
"서울 카페 베스트" 저장 (10곳)
  ↓
다음날 한국 도착
  ↓
"저장한 곳" 리스트 확인
  ↓
순서대로 방문 → 체크인 → 체험권
```

**관광객 (즉흥형)**
```
홍대 도착 → 카페 찾기
  ↓
ZZIK 피드 → 위치 필터 "홍대"
  ↓
최신순/인기순 정렬
  ↓
마음에 드는 곳 선택
  ↓
[지도] 탭 → 경로 안내
  ↓
도착 → 체크인 → 체험권
```

**프랜차이즈 본사 시점**
```
월 50만원 결제
  ↓
옵션 선택:
A. 인플루언서 콘텐츠 (ZZIK 제작)
B. 매장 직원이 직접 업로드
C. UGC 유도 (체험권 제공)

대시보드:
├─ 콘텐츠별 조회수
├─ 저장 수 (관심도)
├─ 실방문 전환율
└─ 체험권 사용률
```

```
Frontend:

Backend:
├─ Node.js + Express
├─ PostgreSQL (콘텐츠 DB)
├─ Elasticsearch (검색/필터)

인프라:
├─ AWS S3 (이미지 저장)
├─ CloudFront (CDN)
├─ Lambda (이미지 리사이즈)
└─ SQS (피드 생성 큐)

비용 예상 (월):
├─ CDN: $200-800 (이미지 위주)
├─ 검색: $100-400 (Elasticsearch)
├─ 저장소: $50-200
└─ 서버: $200-600
총: $550-2,000/월
```

#### 장점 ✅

1. **낮은 진입장벽**
   - 사진 + 텍스트 = 쉬움
   - 기존 小红书 콘텐츠 재활용 가능
   - 빠른 콘텐츠 축적

2. **큐레이션 가능**
   - 품질 통제 쉬움
   - 알고리즘 추천
   - "베스트 콘텐츠" 강조

3. **검색/필터 강점**
   - "성수 카페 데이트"
   - "홍대 맛집 저렴"
   - 의도 기반 탐색

4. **낮은 인프라 비용**
   - 영상 대비 50% 절감
   - 확장 쉬움

5. **B2B 콘텐츠 통제**
   - 본사가 원하는 메시지
   - 일관된 브랜드
   - 프로모션 강조 용이

#### 단점 ❌

1. **차별화 약함**
   - 小红书와 유사
   - "왜 ZZIK인가?" 약함
   - 네트워크 효과 늦음

2. **즉시성 부족**
   - 피드 = 과거 콘텐츠
   - "지금" 느낌 없음
   - 실시간 바이럴 어려움

3. **GPS 검증 필요성 낮음**
   - 피드만 보면 방문 안 해도 됨
   - 체험권 연결 부자연스러움
   - 핵심 기술 가치 ↓

4. **발견 경험 약함**
   - 스크롤 피로
   - "근처 뭐 있지?" 답 안 줌
   - 지도는 부가 기능

5. **바이럴 구조 약함**
   - 피드는 수동적
   - 참여 유도 어려움

## 🎯 전략적 선택 프레임워크

### 질문 1: 핵심 가치는?

```
A. 발견 (Discovery)
   "몰랐던 좋은 곳을 알게 해준다"
   → 피드 플랫폼 유리

B. 경험 (Experience)
   "지금 이 순간을 공유한다"
   → 릴스 지도 유리

C. 전환 (Conversion)
   "실제로 가서 소비하게 한다"
   → 둘 다 가능, GPS 검증이 차별점
```

### 질문 2: 경쟁 우위는?

```
ZZIK의 독점 자산:
1. GPS 무결성 검증 (기술)
2. 본사 발행 체험권 (수익)
3. 외국인 타겟 (시장)

어떤 방향이 자산 활용을 극대화?
├─ 릴스 지도: GPS 검증 = 필수 (자연스러움)
└─ 피드: GPS 검증 = 부가 (어색함)
```

### 질문 3: 시장 타이밍은?

```
현재 트렌드:
├─ 숏폼 영상 급증 (TikTok, Reels, Shorts)
├─ 로컬 콘텐츠 수요 증가 (관광 회복)
└─ 실시간 경험 선호 (라이브 커머스)

5년 후:
├─ 영상이 텍스트를 대체?
├─ AI 큐레이션 보편화?
└─ AR/VR 로컬 경험?

선택:
├─ 릴스: 트렌드 선점, 리스크 높음
└─ 피드: 안정적, 차별화 약함
```

### 질문 4: 실행 난이도는?

```
팀 역량 (가정):
├─ 백엔드: 강함 (GPS 검증 완성)
