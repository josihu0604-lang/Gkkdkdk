## 2️⃣ 전자금융거래법 (2024.9.15 개정 반영) 🚨

### 핵심 변경사항

#### 2024.9.15 개정 내용
```
기존 (2024.9.14까지):
├─ 선불전자지급수단 등록 대상: "2개 업종 + 10개 가맹점"
└─ 비교적 완화된 기준

개정 (2024.9.15~):
├─ 선불전자지급수단 등록 대상: "1개 업종 + 2개 가맹점"
└─ 규제 대폭 강화
```

#### ZZIK 해당 여부 판단 플로우차트

```
┌─────────────────────────────────────────┐
│ Q1. ZZIK이 리워드를 직접 발행하나요?    │
└─────────────┬───────────────────────────┘
              │
         YES ─┼─ NO → 전자지급결제대행업 검토
              │
              ▼
┌─────────────────────────────────────────┐
│ Q2. 연간 총발행액이 500억원 이상인가?   │
└─────────────┬───────────────────────────┘
              │
         NO ──┼─ YES → 6개월 내 등록 필수
              │
              ▼
┌─────────────────────────────────────────┐
│ Q3. 발행잔액이 30억원 미만인가?         │
└─────────────┬───────────────────────────┘
              │
         YES ─┼─ NO → 등록 필요
              │
              ▼
        ✅ 등록 면제 가능
```

#### 면제 기준 (둘 다 충족 필요)
- 발행잔액: 30억원 미만
- 연간 총발행액: 500억원 미만

#### PG 등록 의무 (신규 추가)
```
개정안:
├─ 전자지급결제대행업(PG)으로 등록한 자만 가맹점 계약 체결 가능
├─ 미등록 PG가 가맹점 재정산 진행 시 처벌
└─ 벌칙: 3년 이하의 징역 또는 2천만 원 이하의 벌금
```

#### ZZIK 적용 전략

**시나리오 A: 플랫폼 직접 중개**
```
크리에이터 → ZZIK (중개) → 상점

문제점:
├─ 선불전자지급수단 등록 가능성
├─ PG 등록 필요 가능성
└─ 법률 자문 필수

해결책:
├─ 각 프랜차이즈 본사가 자사 가맹점에만 사용 가능한 쿠폰 발행
├─ ZZIK은 쿠폰 "배포 플랫폼"이지 "발행인" 아님
└─ 분기별 쿠폰 총발행액 내부 모니터링 (임계값: 연 ₩500M)
```

**시나리오 B: 직거래 (권장)**
```
크리에이터 ← ZZIK (매칭만) → 상점

장점:
├─ ZZIK은 매칭 플랫폼
├─ 금융거래 당사자 아님
└─ 등록 의무 없음

구현:
├─ ZZIK은 "만남의 장" 제공
├─ 실제 리워드는 본사가 직접 발행
└─ ZZIK은 수수료만 받음
```

#### 즉시 조치사항 (Week 4까지)

**Week 1-2: 법률 자문**
- [ ] 법무법인에 "ZZIK 자금 흐름 구조" 자문 요청
- [ ] 시나리오 A vs B 비교 분석
- [ ] 등록 면제 해당 여부 정량 기준 확인

**Week 3-4: 법률 구조 확정**

## 3️⃣ 의료법 제27조의3 (환자 유인 등 금지)

### 규제 내용
- 의료기관 광고 시 사전 심의 필수
- 환자 유인 행위 금지 (5년 이하 징역 또는 5천만원 이하 벌금)
- 병원 간 비교, 보증성 광고 금지

### ⚠️ 주의사항 강화

#### 법무법인 자문 필수 사항
```
질의 사항:
1. "플랫폼이 병원명을 지도에 표시하는 것"이 광고에 해당하는가?
2. "체험권 제공"이 환자 유인 금지 조항에 저촉되는가?
3. 보건복지부 2023.08 유권해석의 적용 범위는?
```

#### ZZIK 해결책 (현재 전략 유지)

**✅ 병원명 직접 노출 안 함**
```
앱 내 표시:
❌ "강남 세브란스 병원 - 성형외과 30% 할인"
✅ "강남역 인근 성형외과 - 외국인 친화 시설"
```

**구현 방법:**
1. 지도에 병원 아이콘만 표시 (이름 없음)
2. 클릭 시 "병원 정보" 페이지 (병원 자체 제공 콘텐츠)
3. 쿠폰은 병원이 직접 발행 (ZZIK은 플랫폼만 제공)
4. 광고 아님 명시: "환자 경험 개선 서비스"

#### 법률 검토
- 보건복지부: "플랫폼이 광고주가 아니면 예외" (2023.08 유권해석)
- ⚠️ **주의사항**: 법무법인 자문 시 "환자 유인 금지 해당 여부" 명시 질의 필요

## 4️⃣ 기타소득 과세 (경품 제공 시)

### 규제 내용
- 경품 제공 시 기타소득세 원천징수 의무
- 건별 기타소득금액 5만원 초과 시 과세
- 외국인 비거주자 원천세는 국적·조세조약에 따라 상이

### ZZIK 쿠폰 설계

| Tier | 쿠폰 종류 | 권면가 | 과세 여부 |
|------|----------|--------|----------|
| Tier 1-2 | 커피·햄버거 | ₩5,000~8,000 | ❌ 비과세 |
| Tier 3 | 외식 쿠폰 | ₩30,000 | ❌ 비과세 |
| Tier 4 | 마트 상품권 | **₩49,000** | ❌ 비과세 (안전선) |
| Tier 5+ | 없음 | - | 세무 복잡도 회피 |

### 원천징수 구현

```typescript
// src/utils/tax.util.ts

export const COUPON_TAX_CONFIG = {
  // 과세 최저한
  TAX_THRESHOLD: 50000,
  
  // 안전 마진 (₩50K 경계선 회피)
  SAFE_MAX_VALUE: 49000,
  
  // 원천징수율
  RESIDENT_TAX_RATE: 0.22,  // 20% + 지방세 2%
  
  // 외국인 비거주자 원천세 (조약별)
  NON_RESIDENT_TAX_RATES: {
    'CN': 0.20,  // 중국: 20%
    'JP': 0.20,  // 일본: 20%
    'US': 0.30,  // 미국: 30%
    'DEFAULT': 0.22
  }
};

export function calculateWithholdingTax(
  couponValue: number, 
  userNationality: string,
  isResident: boolean
): number {
  // 과세최저한 이하는 원천징수 없음
  if (couponValue <= COUPON_TAX_CONFIG.TAX_THRESHOLD) {
    return 0;
  }
  
  const taxRate = isResident 
    ? COUPON_TAX_CONFIG.RESIDENT_TAX_RATE
    : (COUPON_TAX_CONFIG.NON_RESIDENT_TAX_RATES[userNationality] 
       || COUPON_TAX_CONFIG.NON_RESIDENT_TAX_RATES.DEFAULT);
  
  return Math.round(couponValue * taxRate);
}
```

### 액션 아이템
- [ ] 쿠폰 권면가 상한 ₩49,000 설정
- [ ] 외국인 원천세 테이블 작성 (중국·일본·미주·유럽)
- [ ] 세무사 자문 (연 1회 지급명세서 제출 요건 확인)

#### 법률 근거
